| R4-001 | resilience | `src/scripts/contact-animation.js:20` | CRITICAL | fixed | `start: "top 85%"` requires the footer's own height to be >= 15% of viewport height for the trigger to ever be crossed, since Contact is the true last element on the page (max reachable scroll caps the footer's top at `(viewport - footerHeight)`). On tall viewports (maximized 4K/16:10 laptops), the footer can fall short of that margin, and the trigger never fires — content stays permanently `opacity:0` since GSAP already hid it via `immediateRender`. Violates the change's own "fail open, not closed" contract. **Fix**: `start` changed to `"top bottom"`, geometrically guaranteed to fire for any visible element. RED confirmed via a 1440x2000 E2E test (opacity stuck at `'0'`) before the fix, GREEN after. `design.md` decision table updated with rationale. |
| R3-001 | reliability | `tests/e2e/visual-refresh.spec.ts:146-159,189-208` | BLOCKER | fixed | Assertions use Playwright's `toBeVisible()`, which ignores `opacity`/`transform` — the exact properties GSAP uses to hide/reveal. These tests pass identically whether ScrollTrigger fires correctly, fires on load, or never fires at all. `design.md:80` explicitly chose this non-discriminating assertion. **Fix**: added a computed-style test asserting `opacity !== '1'` pre-scroll and `opacity === '1'` post-scroll for catalog/FAQ/contact targets; existing `toBeVisible()` checks kept, not replaced. |
